04. CI/CD INTEGRATION

In collaborative team environments, developers often merge Pull Requests directly through the GitHub web user interface. Since local Git hooks do not run during online merges, your repository context files (like CLAUDE.md) can slowly fall out of alignment.

To resolve this, deploy Briefed directly in your CI/CD pipelines. This ensures that every direct push or merge to your main branches automatically triggers a context compilation check and commits updates immediately back to repository files.

🐙 Complete GitHub Action Blueprint

Create a file named .github/workflows/briefed.yml inside your project directory and paste the following workflow script:

name: Briefed Context Update

on:
  push:
    branches:
      - main  # Adjust this to match your default branch name

permissions:
  contents: write

jobs:
  briefed-context:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 2  # Compares previous commit (HEAD^) with current commit (HEAD)

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 18

      - name: Install dependencies
        run: npm ci

      - name: Build project
        run: npm run build

      - name: Set ORIG_HEAD manually
        run: git update-ref ORIG_HEAD $(git rev-parse HEAD~1)

      - name: Run briefed
        run: node dist/cli.js run
        env:
          BRIEFED_API_KEY: ${{ secrets.BRIEFED_API_KEY }}
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Check and commit context changes
        run: |
          if ! git diff --quiet CLAUDE.md; then
            git config --local user.name "github-actions[bot]"
            git config --local user.email "github-actions[bot]@users.noreply.github.com"
            git add CLAUDE.md
            git commit -m "Update AI context [skip ci]"
            git push
          fi

⚙️ Critical Setup Explanations

1. fetch-depth: 2

By default, the GitHub Checkout action performs a shallow fetch (depth of 1) to optimize build speed. Briefed needs to compare commits to generate the summary logs, so you must configure fetch-depth: 2 to fetch the latest commit and its immediate parent.

2. git update-ref ORIG_HEAD $(git rev-parse HEAD~1)

During local merges, Git automatically sets the ORIG_HEAD reference. In CI checkout runners, this reference is absent. We resolve this by running a git reference pointer updates manually, setting ORIG_HEAD to point directly to the parent commit (HEAD~1) before executing the CLI run.

3. [skip ci] in Commit Messages

When the workflow pushes updates back to the repository, it could trigger a recursive loop (the push triggering a new action run, which pushes changes, and so on). Appending the tag [skip ci] (or [ci skip]) inside the commit message instructs GitHub Actions to ignore this push, preventing infinite workflows.

🔑 Configuring GitHub secrets

Ensure your cloud API keys are safely configured in your repository secrets! Go to your repository on GitHub ➡️ **Settings** ➡️ **Secrets and variables** ➡️ **Actions**, click **New repository secret**, and save your credentials under `GEMINI_API_KEY` or `BRIEFED_API_KEY`.
