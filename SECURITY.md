# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Briefed, please **do not** open a public issue. Instead, report it privately to help protect users of this library.

Please send vulnerability reports via email to **chaitanya.anand@outlook.com**.

### What to include:
* A detailed description of the vulnerability.
* Steps to reproduce the issue (proof-of-concept script/commands).
* Potential impact of the vulnerability.

We will review your report, request additional details if needed, and coordinate a patch release as quickly as possible. Thank you for helping keep Briefed secure!
